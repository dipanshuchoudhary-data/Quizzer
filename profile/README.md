## 🧑‍💻 About Me

I specialize in building **AI-native applications** — from LangGraph-orchestrated agentic pipelines to RAG systems that make large knowledge bases actually useful. My work sits at the intersection of software engineering discipline and applied ML: I care as much about system reliability and clean architecture as I do about model quality.

```text
🤖  Agentic AI systems    ──  multi-agent architectures, tool-calling, autonomous task execution
🔍  LLM applications      ──  RAG, structured output, prompt engineering, LLM evaluation
⚙️  Production AI backends ──  async APIs, background job pipelines, scalable inference
```

- 🔭 Currently building **multi-agent orchestration** systems with LangGraph
- 🌱 Deep-diving into **LLM fine-tuning**, **LLM-as-judge evaluation**, and **hybrid RAG** architectures
- 💬 Ask me about **FastAPI**, **LangChain**, **Agentic AI**, or **production ML systems**
- ⚡ Fun fact: I think good software architecture and good AI systems share the same first principle — *clarity of purpose*

### 🔍 DocuMind
**Enterprise RAG System**

> Organizations have thousands of documents and no way to query them reliably. Generic chatbots hallucinate. Keyword search misses context.

**✨ Key Features:**
- 🔀 Hybrid retrieval: dense vector + BM25, re-ranked by cross-encoder
- 💡 HyDE + query rewriting for better recall on vague queries
- 📎 Source citations on every answer — no hallucination without accountability
- 🌊 Streaming responses with conversation history

### 🧪 LLM Eval Bench
**Automated LLM Evaluation Framework**

> Prompts break silently. Model updates change behavior. There's no safe way to upgrade without a regression test suite.

**✨ Key Features:**
- 📋 YAML-based test cases: input, expected output, criteria
- ⚖️ Evaluators: exact match, semantic similarity, LLM-as-judge, regex
- 🔄 CI-compatible — runs on every prompt or model change
- 📈 Metrics dashboard: pass rate, latency, token cost
